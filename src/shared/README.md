# Shared State and Synchronization (Student Work Area)

## Purpose

This directory contains shared data structures and synchronization logic introduced in **Module 5**.

Shared state must be protected to prevent race conditions and data corruption.

---

## What Students Should Do

- Define shared data structures
- Protect shared access using mutexes or semaphores
- Demonstrate correct behavior under concurrent access

---

## Evidence Expectations

- Stable output under concurrent access
- Optional before/after comparison (unprotected vs protected)
- Notes justifying synchronization choices
