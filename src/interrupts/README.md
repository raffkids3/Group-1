# Event-Driven and Interrupt Logic (Student Work Area)

## Purpose

This directory contains event-driven logic introduced in **Module 7**.

Interrupts and signals represent asynchronous events that must be handled safely.

---

## What Students Should Do

- Implement ISR-safe signaling mechanisms
- Defer work from interrupt context to task context
- Keep ISR code minimal and bounded

---

## Evidence Expectations

- Output showing event trigger and task-side handling
- Notes explaining ISR discipline and responsiveness impact
