# Timing and Watchdogs (Student Work Area)

## Purpose

This directory contains timing-related logic introduced in **Module 4**.

Timing behavior is a first-class requirement in real-time systems.

---

## What Students Should Do

- Implement periodic behavior using RTOS timers or delays
- Add watchdog-style logic to detect delayed or stalled execution
- Make timing behavior observable (timestamps, counters)

---

## Evidence Expectations

- Output showing periodic execution
- Output showing watchdog detection under fault conditions
- Notes explaining normal vs abnormal timing behavior
